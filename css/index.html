定位

  1. 定位（重点）
    • 作用：改变元素位置
    • 分类：
      • 静态定位
      • 绝对定位
      • 相对定位
      • 固定定位
      1. 静态定位：positin:static;
          总结：
        1. 元素无法通过静态定位的方式移动位置
        2. 静态定位的元素就是标准流的元素
      2. 绝对定位：positin:absolute;
        • 总结：
          1. 通过绝对定位可以改变元素的位置
          2. 如果父元素没哟设置定位或者设置了静态定位。那么父元素中的的绝对定位的子元素，该子元素是以浏览器可视区域左上角为参照进行移动。
          3. 如果父元素设置了除静态定位以外的其他定位，那么该绝对定位子元素会以父元素左上角为参照进行位置移动。
          4. 绝对定位的元素不占位置（脱标）
          5. 绝对定位可以实现模式转换
        • 使用场景：
          一般在网页布局中如果出现盒子压盒子（标签压标签）的效果时，该考虑用绝对定位。
  2. 相对定位:position:relative;
    1. 总结：
      1. 相对定位可以改变元素位置
      2. 相对定位的元素始终是相对于元素原来的位置进行移动
      3. 相对定位的元素没有脱标。占位置
      4. 相对定位的元素不能实现模式转换
    2. 使用场景：
      1. 如果一个子元素设置了绝对定位，那么其父元素一般设置相对定位（子绝父相）
  3. 固定定位：position:fixed;
    总结：
    1. 通过固定定位可以移动元素位置
    2. 固定定位的元素始终是以body（浏览器）可视区域左上角为参照进行位置移动
    3. 固定定位的元素不占位置（脱标）
    4. 固定定位可以实现模式转换
  • 定位层级关系（重点）
    1. 层级特点只适用于定位（除静态定位） 的元素
    2. 通过z-index属性可以改变元素的层级关系
    3. z-index层级关系总结：
      1. 一个元素设置了（除静态定位）定位，那么该元素的z-index默认值就是auto。
      2. 定位的元素层级高于标准流的元素和浮动的元素。
      3. 如果z-index值越大，那么层级越高。
      4. 如果z-index值相同，那么最后的定位元素会压着前面的定位元素（后来居上）。
      5. 如果父元素设置了层级关系，那么父元素的z-index值越大，子元素的值就大。
  • 定位居中：
    1. margin: 0 auto;只能让标准流的元素居中
    2. 总结： 先通过left：50%先走父元素的一半
      通过margin-left：负的自己的一半

    • 浮动及特点（重点）
      • 属性：float:left|right
      • 作用：
        • 图片环绕文字的效果（浮动出现的最原始的目的）
        • 网页布局（让块级元素在一行上显示）
        • 网页导航制作（让块级元素在一行上显示）
      • 特点：
        • 浮动可以让块级元素在一行上显示（图标）
        • 设置完浮动都该元素不占位置（脱标）
        • 浮动的元素只会影响其后面的元素
        • 如果要让元素在一行上显示（块级元素）， 那么这些元素都要设置浮动
        • 浮动可以实现模式转换（脱标）
          注意：如果要让元素实现模式转换，只能使用display。
    • 浮动作用（使用场景）
      • 实现图片环绕文字效果（图片设置浮动）
        文字不会受浮动的影响
      • 在网页布局中（画盒子），要实现块级元素（div）在一行上显示，那么就设置浮动。
    • 清除浮动（重点）
      • 清除浮动：
        清除浮动的元素对后面元素的影响
      • 删除浮动:不使用浮动
      • 方式：
        • 使用clear属性清除浮动。先使用空标签放在需要清除浮动的后面。clear:both；全部清除
        • 给父元素设置overflow:hidden;
          • 注意：
            如果父元素中没有定位的元素，那么可以使用该属性，反之不推荐
        • 使用伪元素去清除浮动：
          .clearfx：after{ content:"";
          display:block;
          height:0;
          line-height:0;
          clear:both;
          visibility:hidden; }
          .wrapper{ zoom:1; }兼容IE浏览器
        • 总结：
          • 什么时候需要清除浮动？
              如果一个父元素没有设置高度，而且该父元素中的所有子元素都设置了浮动。那么后面的元素如果跑到了父元素的下面====》，这个时候我们考虑只用浮动。
